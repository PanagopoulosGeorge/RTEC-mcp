Two Core Capabilities

1. Rule Generation P1 Complete

Turn a natural-language description of a composite activity into valid RTEC Prolog rules.

  • Stateless generator agent
  • Deterministic grounding from rule heads

2. Querying P2 Deferred

Turn NL questions about recognition results into queries executed against RTEC output.

  • Activity/entity/time resolution
  • holdsAt/holdsFor query dispatch
  • Grounded answer generation

The Central Thesis Contribution

The scientific contribution is NOT "an LLM can write Prolog."

It is: execution-guided iterative self-repair produces measurable improvement in rule quality, with RTEC execution against ground-truth intervals as the behavioral oracle.

EDF vs SDF: The Central Asymmetry

EDF Event-Driven Fluents

Defined by initiatedAt/terminatedAt rules over discrete events.

initiatedAt(withinArea(V, AT)=true, T) :-
    happensAt(entersArea(V, Area), T),
    areaType(Area, AT).

✓ Converges fast (often single-shot)

SDF Statically Determined Fluents

Defined by a single holdsFor rule using interval algebra.

holdsFor(loitering(V)=true, I) :-
    holdsFor(lowSpeed(V)=true, Il),
    holdsFor(stopped(V)=farFromPorts, Is),
    union_all([Il, Is], Iu),
    intDurGreater(Iu, Thresh, I).

✗ Plateaus below convergence

⚠️ The #1 Problem: Compilable ≠ Correct. The LLM produces Prolog that compiles cleanly but recognizes the wrong intervals. A clean compile tells you NOTHING about behavioral correctness. The only valid oracle is RTEC execution scored against ground-truth intervals.

🔌 Adapters

Concrete implementations of port interfaces

Engine Adapter: RtecSubprocessEngine

Implemented rtec_subprocess.py

Runs the REAL RTEC engine via SWI-Prolog subprocess.

Two-Step Invocation

1. Compile
compile.sh --event-description=rules.prolog
→
2. Run
swipl -l continuousQueries.prolog -g continuousQueries(...)

Key Design Decisions

  • Separates compile-time from run-time failures into distinct typed errors
  • Avoids the driver's sleep 10 on failed compile
  • Writes only to temp directories — never modifies examples/
  • Parses recognised intervals directly (no file re-read)
engine = RtecSubprocessEngine(
    app="maritime",
    repo_root=Path("/path/to/RTEC-agent"),
    timeout_s=600.0
)
result = engine.run(
    rules=prolog_rules,
    declarations=grounding_decls,
    event_stream=Path("examples/maritime/dataset/csv/brest_critical.csv"),
    window=Window(start_time=1443650400, end_time=1443686400, window_size=36000, step=36000),
    static_data=(aux / "compare.prolog", aux / "loadStaticData.prolog"),
)

Vocabulary Structure

A Vocabulary is the closed set of symbols an LLM is allowed to emit. Every event, fluent, value, and threshold the generator produces must exist here. Inventing a plausible-but-absent predicate is the #1 failure mode.

Predicate

Input event signature

name: "entersArea"
arg_names: [vessel, area]
meaning: "Vessel enters area"

name: "withinArea"
arg_names: [vessel, areaType]
values: ["true"]
fluent_type: EDF

BKPredicate

Background knowledge

name: "inRange"
arg_names: [value, min, max]
note: "EXCLUSIVE comparator!"

ThresholdKey

Named threshold parameter

key: "loiteringTime"
value: "1800"
meaning: "Min duration (s)"

ValueDomain

Closed enumeration of atoms

  natura, nearCoast, ...]

Entity grounding info

arg_names: [vessel, vessel1, ...]

System Prompt Structure

RTEC Primer

  • Event Calculus conventions
  • Core predicates: happensAt, holdsAt, holdsFor, initiatedAt, terminatedAt
  • EDF vs SDF regime decision heuristic
  • Hard rules: vocabulary enforcement, no grounding declarations

Vocabulary Rendering

VOCABULARY for domain `msa` (closed set — use only these):

Input events (happensAt):
  - entersArea(vessel, area) — Vessel enters a geographic area
  - leavesArea(vessel, area) — Vessel leaves a geographic area
  ...

Output fluents (what rules may define or reference):
  - withinArea(vessel, areaType)={true} [EDF] — Vessel is inside area
  ...

Background-knowledge predicates:
  - areaType(area, type) — Maps area id to type
  - inRange(value, min, max) — True when Min < Value < Max
    [NOTE: EXCLUSIVE comparator — disagrees at endpoints!]
  ...

Threshold keys (use as thresholds(Key, Value)):
  - loiteringTime (= 1800) — Minimum duration (seconds)

Grounding Generation

CLAUDE.md §5 Invariant 4: Grounding declarations are derived deterministically from the rule head, never authored by the LLM. Missing grounding is the #1 silent failure mode — generating it ourselves removes that whole failure class.

Input: Rule Head

initiatedAt(withinArea(Vessel, AreaType)=true, T) :-
    happensAt(entersArea(Vessel, Area), T),
    areaType(Area, AreaType).

Output: Grounding

grounding(withinArea(Vessel, AreaType)=true) :-
    vessel(Vessel),
    areaType(AreaType).

Grounding Rules

  • Entity arguments (vessel, vessel1, vessel2) → entity predicate (vessel/1)
  • Two entity args + pair_predicate → vpair/2 instead of cross-product
  • Value domain arguments → domain predicate (areaType/1)
  • Boolean value (true) → literal, no goal

Components

RepairState P2

TypedDict threaded through every node

  • Current rule
  • Latest ExecutionResult and ScoreResult
  • Best-so-far snapshot
  • Typed repair feedback
  • Iteration counter

Orchestrator P2

Stateful agent for diagnosis and routing

  • Diagnoses failures from execution diff
  • Issues typed repair instruction
  • Performs deterministic routing
  • Prioritises reasoning over output-form

Feedback Builder P2

Maps disagreements to cause classes

  • Boundary comparator
  • Wrong termination
  • Missing branch
  • Off-by-one

Critical Invariants

⚠️ Best-So-Far Policy

F1 across iterations is NOT guaranteed monotonic. Only commit a repaired rule when its F1 strictly improves on the best seen so far; otherwise keep the previous best.

⚠️ No Vague Feedback

The repair instruction must be structured and specific. Vague "try again" signals are forbidden. Feed back the disagreement timestamps mapped to a cause class.

The Four Silent RTEC Failure Modes

1. Missing Grounding

Every authored fluent needs grounding declared, or it silently produces nothing.

✓ Solved by deterministic generation

2. Wrong Fluent Regime

EDF modeled as SDF, or vice versa.

⚠ Requires semantic diagnosis

3. Unbound Under Negation

Variable appearing only inside not(...) is unsafe.

⚠ Static check possible

4. Off-by-One

Effects take hold at T+1, not T. Endpoints drift by one tick.

✗ Requires interval analysis

Ground Truth Format

Ground-truth files use raw RTEC recognitions(...) output format with half-open [s, e) intervals. A drift here is invisible at parse time and silently inflates FN/FP.

recognitions(predictions,withinArea,[[vessel_1,nearCoast],true],[(1443650400,1443652800)]).
recognitions(predictions,withinArea,[[vessel_2,fishing],true],[(1443651000,1443655000),(1443660000,1443665000)]).
